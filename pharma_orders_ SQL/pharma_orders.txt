
## SQL_1: Топ-3 аптеки по объёму продаж 

###  Описание задачи
Определить три аптеки с наибольшим объёмом продаж.  
Объём продаж рассчитывается как **цена × количество** для каждой позиции заказа, затем суммируется по аптекам.

###  Используемые SQL-инструменты
- `SELECT`
- `SUM()` — агрегирующая функция для подсчёта общей суммы
- `GROUP BY` — группировка по названию аптеки
- `ORDER BY ... DESC` — сортировка по убыванию
- `LIMIT 3` — ограничение вывода до трёх записей
- Арифметическая операция: `price * count`

###  Вывод
Топ-3 аптеки по выручке:
1. **Столичная** — 1 896 438 ₽  
2. **Доктор Айболит** — 1 893 776 ₽  
3. **Здравсити** — 1 863 037 ₽  

 Эти аптеки показывают наибольшую выручку. Желательно:
- Изучить их ассортимент и маркетинговые практики
- Рассмотреть масштабирование успешных решений

## SQL_4: Накопленная сумма продаж по аптекам во времени

### Описание задачи
Рассчёт накопленной суммы продаж для каждой аптеки нарастающим итогом по дням.

### Используемые SQL-инструменты
Агрегация: SUM(price * count) → дневная выручка
Группировка: GROUP BY pharmacy_name, report_date
Оконная функция:
SUM(SUM(price * count)) OVER (PARTITION BY pharmacy_name ORDER BY report_date ROWS UNBOUNDED PRECEDING)
Таблица: pharma_orders

### Результат  
получаем 4 поля:
pharmacy_name — название аптеки
report_date — дата заказа
daily_sales — объём продаж за этот день (price × count)
cumulative_sales — общая сумма продаж с начала наблюдений по эту дату включительно


### Вывод 
Отчёт показывает, рост выручки каждой аптеки с течением времени.
- Каждая аптека имеет свою траекторию роста выручки: одни быстро набирают оборот, 
другие — стабильно, но медленнее.
Накопленная сумма позволяет:
  - Объективно сравнивать точки независимо от даты открытия или сезонных всплесков
  - Выявлять отклонения от ожидаемой динамики 
  - Оценивать эффективность управленческих решений 
Этот анализ служит основой для:
- Еженедельного мониторинга операционной эффективности сети
- Формирования персонализированных KPI для управляющих
- Прогнозирования достижения плановых показателей по каждой точке

## SQL_8: Топ-10 самых частых клиентов в аптеках «Горздрав» и «Здравсити» 

### 📌 Описание задачи
Выявить 10 клиентов с наибольшим количеством заказов в каждой из аптек: «Горздрав», «Здравсити».
Для этого:
- Созданы две временные таблицы (CTE) — по одной на аптеку  
- Внутри каждой — соединение с таблицей клиентов, подсчёт числа заказов  
- Клиенты ранжированы и отобраны топ-10  
- Результаты объединены через `UNION ALL`

### Используемые SQL-инструменты
- `WITH ... AS (...)` — два CTE для разделения логики
- `JOIN` — связь заказов и клиентов
- `COUNT(*)` — подсчёт частоты заказов
- `ROW_NUMBER() OVER (ORDER BY COUNT(*) DESC)` — ранжирование
- `WHERE pharmacy_name = '...'` — фильтрация по аптеке
- `UNION ALL` — объединение результатов
- `CONCAT_WS` — корректное объединение ФИО с обработкой NULL
- Таблицы: `pharma_orders`, `customers`
 
 ### Результат:
- Полное ФИО клиента
- Название аптеки
- Количество заказов

### Вывод
В аптеках «Горздрав» и «Здравсити» выявлены клиенты, совершившие наибольшее число заказов.    
В «Горздрав»:  
 — Петров Иван Сергеевич — 28 заказов  
— Сидорова Анна — 25 заказов
В «Здравсити»:  
— Кузнецов Дмитрий — 31 заказ  
— Орлова Елена — 29 заказов  
Высокая частота заказов указывает на:
- Регулярные закупки препаратов,
- Доверие к конкретной аптеке,
- Удобство сервиса.  
Возможно: 
- Персональное обслуживание: запомнить предпочтения, уведомлять о поступлении нужных препаратов.
- Сравнение поведения: почему в «Здравсити» топ-клиенты делают больше заказов? Может, лучше сервис?
